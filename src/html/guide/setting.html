환경세팅

퍼블리싱 기본 환경 세팅 가이드 입니다.

NPM

  • node 버전은 14.17.6 기준으로 설정 (14.xx 버전으로 설치)
  • package.json 기준으로 설치
  • gulp 는 4.X 기준으로 작성되었습니다.

GULP TASK

  • watch_web - 로컬 서버 시작 및 관련 리소스 watch
  • build - src 관련 리소스를 구축하여 dist에 전달
  • min_js - src/js 폴더의 js 확장자를 min 파일로 압축 dist에 전달
  • bs - 브라우저싱크 로컬서버 시작
  • spriteIcon - src/images/sprite/ 폴더내의 이미지를 dist/images/sprite.png 파일로 취합해줌 (폴더생성시 폴더명으로 sprite.png 파일을 분리해줌)
  • datauri - src/images/datauri/ 폴더내의 이미지를 이미지파일명으로 class로 mixin 으로 dataIMG로 변환
  • sass - src/sass/ 폴더내의 scss 파일을 취합 css로 변환
  • imageMin - src/images/ 폴더안의 이미지파일을 간소화시켜 dist/images 에 배포 (gif,jpg,png,svg) sprite.png 는 예외
  • imgUpdate - src/images/ 폴더안의 이미지파일은 단순 복사하여 dist/images/ 로 이동
  • html - src/html/ 폴더안의 html 파일을 취합하여 dist/html에 전달 /*/include 파일은 제외
  • htmlPUG - src/html/pug/ 폴더안의 *.pug 파일을 html로 변환하여 dist/html에 변환

그외

  • error 나는 대부분의 이유는 node 버전때문이며 node.js 기존버전을 병합하여 사용해야할경우 nvm 을 별도로 설치하여 node.js 버전변경 하여 사용
  • 특정 패키지에서 지속되는 에러가 발생시 설치후 정상적인 사용자의 node_modules 폴더를 덮어씌워 사용
